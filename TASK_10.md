# TASK 10 — Match overlay replacing the inline match panel

> Conventions and commands: see the backlog index (Claude plan file). Run everything via `mise exec -- pnpm <cmd>`, never commit. Web tests mock the store module (`vi.mock('@/store/game-store', ...)`) — new store fields must be added to affected component test mocks. Update `MANUAL_TEST_PLAN.md` when this task finishes.

**Context**: today `MatchTab.tsx` renders pre-match info (header, two `FormationGrid` lineups, form, `TacticsSection`) with `MatchSimPanel` appended below — a long scrolling column where live output (ticker, stats, subs) sits below the fold. Replace the live/completed presentation with a near-fullscreen **modal overlay** that holds everything needed for in-game tactical decisions. It must read unmistakably as a modal, and it closes when the user clicks Next match / Continue after full time. This overlay is also the future home of mid-match formation changes (deferred TASK_02). **User decision: the overlay opens for BOTH Play Match (live) and Simulate (final result view).**

**Current structure (verified)**: `MatchSimPanel.tsx` (255 lines) = scoreboard header + phase/status line + mode-dependent controls (pause / resume+skip / play+simulate+sim-season / next-match+sim-season) + XI-blocked & red-card & injury `Alert`s + `SubstitutionPanel` (paused, live) + `MatchInsightCards` (HT read, FT analysis) + `MatchStatsSheet` (live + FT) + event ticker (`EventItem` list, `maxHeight: 320`). Store fields it reads: `focusFixture, focusLive, matchEvents, isStreaming, pauseRequested, pausesUsed, lastPauseReason, streamHome/Away/Minute, lastMatchInsights, lastMatchStatistics, halfTimeInsights` + actions `advanceMatch, pauseMatch, skipMatch, goToNextMatch, simulateToEnd`.

**Build**:
- **Store** (`apps/web/src/store/game-store.ts`): new `matchOverlayOpen: boolean`. Set `true` in `advanceMatch` and `skipMatch` (the player's fixture starting/being simulated); set `false` in `goToNextMatch` and via a new `closeMatchOverlay()` action. Not persisted.
- **New `apps/web/src/components/match/MatchOverlay.tsx`**: MUI `Dialog` — `fullWidth`, `maxWidth={false}`, paper sx `height: 'calc(100vh - 48px)', m: 3` (visible backdrop margin on md+ makes the modal-ness obvious), `fullScreen` on xs. While the match is live (not completed): `disableEscapeKeyDown`, backdrop click ignored — the only ways forward are the match controls. Once `focusFixture.status === 'completed'`: "Next match" (existing `goToNextMatch`) closes it; also show a Continue/close affordance and keep "Sim. Season" (through the TASK 8 confirm modal).
- **Layout inside** (two panes on md+, stacked on xs): sticky top bar = scoreboard + minute/phase + controls (reuse the existing header/controls JSX) + the red-card/injury alerts. Left pane = event ticker filling the available height (flex, `overflowY: auto`, auto-scroll to newest) — this kills the scrolling complaint. Right pane = `TacticsSection` (style + sliders, `disabled={isStreaming}` — moves INTO the overlay so mid-match tactics live where the match is) + `SubstitutionPanel` (paused, live) + `MatchStatsSheet` + `MatchInsightCards` (HT read while paused at the interval; FT analysis when completed). Leave an obvious slot in the right pane for the future formation control (TASK_02).

Reuse `EventItem`, `MatchStatsSheet`, `MatchInsightCards`, `SubstitutionPanel`, `resolvePlayer` as-is (move/share, don't rewrite). `MatchTab` keeps its pre-match content (competition header, lineups, form, tactics) unchanged.
